# Protokollanalyse

## Netzwerk Topologien

- Direktverbindung
    - Einfache Implementierung
    - Hohe Übertragunsgeschwindigkeit
    - Weniger Störungen
    - Begrenzte Skalierbarkeit
    - Hohe kosten bei Erweiterung
    - Anfälligkeit für Störungen

- Busverbindung
    - Kosteneffizient aufgrund weniger Kabelbedarf
    - Einfach zu Implementieren
    - Geringer Platzbedarf
    - Begrenzte Länge des Busses
    - Leistungsprobleme der Bandbreite
    - Fehleranfällig

- Sternberbindung
    - Hohe Leistung
    - Einfache Fehlerdiagnose und Verwaltung
    - Geringer Hardwarebedarf
    - Einfache Erweiterung
    - Zentrale Abhängikeit zum Switch
    - Höhere Kosten für Hardware (Switch)
    - Verkabelungsaufwand
    - Leitungsbottleneck beim Switch

## Internet-Architektur

Ein Netzwerk besteht typischerweise aus:
- Clients (PCs)
- Switches (verbinden Geräte im lokalen Netzwerk)
- Routern (verbinden verschiedene Netzwerke)
- Servern

Wichtige Konzepte:
- IP-Adresse → eindeutige Adresse eines Geräts
- Subnetzmaske → definiert Netzwerkbereiche
- Default Gateway → Router zum Internet

## OSI-Modell

Das OSI-Modell beschreibt Netzwerkkommunikation in 7 Schichten:

- Bitübertragung (Physical)
- Sicherung (Data Link)
- Vermittlung (Network)
- Transport
- Sitzung
- Darstellung
- Anwendung

Beispiele für Protokolle:
- Ethernet / ARP
- IP
- TCP / UDP
- HTTP / DNS / FTP

## Netzwerkprotokolle

Protokolle sind Regeln für die Kommunikation zwischen Computern. Sie definieren:

- Format der Nachrichten
- Bedeutung der Daten
- Reihenfolge der Kommunikation
- Fehlerbehandlung

Ohne Standards wären Systeme verschiedener Hersteller nicht kompatibel.

## Netzwerk-Analyse mit Wireshark

Wireshark ist ein Tool zum Mitschneiden und Analysieren von Netzwerkpaketen. Mögliche Anwendungen:

- Fehleranalyse im Netzwerk
- Untersuchung von Netzwerkverkehr
- Erkennung von Angriffen
- Analyse von Protokollen

Beispiel im Labor:

1. ping 8.8.8.8 ausführen
2. Pakete in Wireshark filtern (ip.addr == 8.8.8.8)
3. ICMP-Pakete analysieren

## Typische Netzwerkangriffe

Mögliche Angriffe auf Netzwerke:

- Sniffing – Abhören von Netzwerkverkehr
- Manipulation – Veränderung von Daten
- Maskerading – falsche Identität vortäuschen
- Replay – Wiederholen von Nachrichten
- Denial of Service (DoS) – Netzwerk lahmlegen
- Hijacking – Übernahme einer Verbindung
